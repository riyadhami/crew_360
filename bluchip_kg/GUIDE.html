BluChip 360 Knowledge Graph

IndiGo BluChip member intelligence — setup guide & architecture reference

◆ What this builds

A persistent member 360 knowledge graph stored in Azure Cosmos DB (Gremlin API). It turns flat booking data, partnership earn records, and behavioural attributes into a connected graph that agents and LLMs can traverse to understand a BluChip member holistically.

Question an agent can answerHow the graph answers it
What tier is this member and what benefits do they have?Member → BELONGS_TO_TIER → Tier
Are they a high-frequency flyer or a one-time visitor?Member.flyer_frequency
Which partners do they earn BluChips with?Member → EARNS_POINTS_FROM → PartnerTransaction → WITH_PARTNER → Partner
Do they prefer DEL→BOM or BOM→HYD?Member → PREFERS_ROUTE → Route
Are they a business or leisure traveller?Member.travel_purpose
What fare type do they mostly book?Member → USES_FARE_TYPE → FareType
What add-ons does this member typically buy?Member → MADE_BOOKING → Booking → CONTAINS_SEGMENT → Segment → PURCHASED_ADDON → AddOn
Show me all flights this member took last yearMember → MADE_BOOKING → Booking → CONTAINS_SEGMENT → Segment.flight_date
ℹ️ The graph is built once and stored permanently in Cosmos DB. Agents only ever read it — nothing is created or destroyed during a query.

◆ Ontology

Ten vertex types, eleven edge types. Transactional / event-driven design — individual Booking, Segment, PartnerTransaction, and AddOn nodes replace the old aggregated TravelProfile. Behavioural summaries are properties on the Member vertex; the full transaction history lives in the graph.

── Vertex Labels ──────────────────────────────────────────────────────

  Member               FFN ID · demographics · spending_segment · travel_purpose · flyer_frequency
  Tier                 Blu 3 (entry) / Blu 2 / Blu 1 (top) · spend+flight thresholds · earn rates · perks
  Booking              Per PNR · booking_date · channel · fare_type · status · trip_type · total_fare
  Segment              Per flight leg · flight_number · date · seat · meal · baggage · travel_purpose
  PartnerTransaction   Individual earn event · bluchips_earned · spend_amount · activity_date
  AddOn                Catalog node · XL Seat / Fast Forward / Meal / Baggage / 6E Prime …
  Partner              19 partners · earn rate · category (FINANCIAL / NON_FINANCIAL)
  Route                Origin→Destination pair · shared across members · DOM / INTL
  Airport              15 airports · METRO / LEISURE / INTERNATIONAL · region
  FareType             8 fare classes: SAVER / FLEXI / CORPORATE / SME / SUPER6E / STRETCH / PROMO / RETURN_FAMILY
  Airport         Code · city · type (METRO / LEISURE / INTERNATIONAL)
  FareType        SAVER / FLEXI / CORPORATE / SME / SUPER6E / STRETCH / PROMO …


── Edge Labels ────────────────────────────────────────────────────────

  (Member)             ──[BELONGS_TO_TIER   {bluchips_balance, qualifying_spend, qualifying_flights, tier_since}]──▶ (Tier)
  (Member)             ──[MADE_BOOKING                                                                          ]──▶ (Booking)
  (Booking)            ──[CONTAINS_SEGMENT                                                                      ]──▶ (Segment)
  (Segment)            ──[FLIES_ON                                                                              ]──▶ (Route)
  (Route)              ──[ORIGIN_AIRPORT                                                                        ]──▶ (Airport)
  (Route)              ──[DEST_AIRPORT                                                                          ]──▶ (Airport)
  (Segment)            ──[PURCHASED_ADDON                                                                       ]──▶ (AddOn)
  (Member)             ──[EARNS_POINTS_FROM                                                                     ]──▶ (PartnerTransaction)
  (PartnerTransaction) ──[WITH_PARTNER                                                                          ]──▶ (Partner)
  (Member)             ──[USES_FARE_TYPE    {usage_count, pct_of_bookings}                                      ]──▶ (FareType)
  (Member)             ──[PREFERS_ROUTE     {trip_count, last_flown}                                            ]──▶ (Route)
  (Route)  ──[ARRIVES_AT                                                 ]──▶ (Airport)
ℹ️ Tiers are based on flight spend + flight count in the last 12 months — not on BluChip balance. BluChips balance is a separate redeemable currency that accumulates independently of tier status.

◆ Pipeline

There are two completely separate phases. The build phase runs once. The query phase runs on every agent call.

Build phase (run once)

Static reference data (tiers, partners, fare types, airports)
          │
          ▼  setup_azure.py  ──  provisions Cosmos DB account + graph container
          │
          ▼  graph_builder.py --static-only  ──  loads reference vertices
          │
          ▼  synthetic_generator.py  ──  generates realistic member data
             (covers every column from excel_datapoints.txt)
          │
          ▼  graph_builder.py --members N  ──  for each member creates:
             · Member vertex
             · BELONGS_TO_TIER edge  (Member → Tier)
             · MADE_BOOKING edges    (one per Booking vertex)
             · CONTAINS_SEGMENT edges (Booking → Segment)
             · FLIES_ON edges         (Segment → Route)
             · ORIGIN_AIRPORT / DEST_AIRPORT edges  (Route → Airport)
             · PURCHASED_ADDON edges  (Segment → AddOn catalog nodes)
             · EARNS_POINTS_FROM edges (Member → PartnerTransaction)
             · WITH_PARTNER edges     (PartnerTransaction → Partner)
             · USES_FARE_TYPE edges   (Member → FareType)
             · PREFERS_ROUTE edges    (Member → Route)
             · USES_FARE edges
          │
          ▼  Cosmos DB (Gremlin)  ──  persistent graph, stays forever

Query phase (every agent call)

User / LLM question
          │
          ▼  BluChipGraphDB method called  (cosmos_graph_traversal.py)
          │
          ▼  Gremlin READ query  ──  no writes, graph unchanged
          │
          ▼  Python dict returned  ──  agent synthesises answer
✅ Agents never modify the graph. The graph is only changed by re-running graph_builder.py when your source data changes.

◆ File structure

bluchip_kg/
  setup_azure.py           ← run FIRST — provisions all Azure resources + writes .env
  .env.example             ← template; auto-filled by setup_azure.py
  requirements.txt
  src/
    graph_builder.py        ← main pipeline — loads all vertices + edges into Cosmos DB
    data/
      static_data.py         ← all tiers, 19 partners, 8 fare types, 15 airports
      synthetic_generator.py  ← generates realistic members (all 100+ datapoints)
    utils/
      cosmos_helpers.py      ← Gremlin client, retry logic, vertex/edge builders
      cosmos_graph_traversal.py ← BluChipGraphDB — all LLM-callable read queries

◆ Query layer

All Gremlin queries live in BluChipGraphDB. These are the methods you expose as tools to an LLM agent.

MethodWhat it returns
Catalogue / Overview
schema()Vertex label counts, all edge types, total edge count
tier_overview()All tiers with earn rates, upgrade thresholds, live member counts
partner_overview()All partners with earn rate, category, transaction count
addon_catalog()All available add-on types with category and price
Member-centric tools
member_profile(ffn_id)Full 360 view — demographics, tier, preferred routes, fare usage, booking/transaction counts
member_bookings(ffn_id, limit)Booking list with channel, fare, status, segment count, total fare
booking_details(booking_id)All segments in a booking + add-ons per segment
segment_history(ffn_id, limit)Flat list of all flight segments ever flown by a member
member_addons(ffn_id)Add-on purchase counts across all segments
member_partner_transactions(ffn_id)All earn events with partner name, spend, BluChips
member_preferred_routes(ffn_id)Routes with trip count + last flown date (PREFERS_ROUTE edges)
Population / Segment tools
tier_members(tier_name)All members on Blu 3 / Blu 2 / Blu 1 with qualifying metrics
route_travelers(origin, dest)Members who prefer a specific route (PREFERS_ROUTE)
popular_routes(top_n)Routes ranked by number of members + segment count
partner_transactions(partner_name)All earn transactions for a partner with member FFNs
members_by_fare(fare_name)Members who use a fare type (USES_FARE_TYPE edges)
members_by_addon(addon_type)Members who purchased a specific add-on + count
high_value_members(segment)Members matching HIGH_FREQ_HIGH_SPEND / LOW_FREQ_HIGH_SPEND etc.
segment_purchase_summary(origin, dest)Route-level add-on + fare breakdown from real segment data
member_subgraph(ffn_id, depth)N-hop neighbourhood for full contextual LLM view

1 Install prerequisites

You need two things on your machine before anything else.

Python 3.11+

Check if you already have it:

powershellpython --version

If not → python.org/downloads — check "Add to PATH" during install.

Azure CLI

Check if you already have it:

powershellaz --version

If not → aka.ms/installazurecli — download the Windows MSI and run it.

2 Navigate to the project folder

powershellcd "C:\Random\Knowledge Layer\Indigo_Knowledge_Layer-main\bluchip_kg"

3 Create a virtual environment

Always use a venv so packages don't clash with other projects on your machine.

powershellpython -m venv venv
venv\Scripts\activate
✅ You'll see (venv) appear at the start of your terminal line. Every command from here must be run with the venv active.

4 Install dependencies

powershellpip install -r requirements.txt

This installs:

PackagePurpose
gremlinpythonGremlin client — talks to Cosmos DB over WebSocket
python-dotenvLoads your .env credentials file automatically

5 Login to Azure

powershellaz login

This opens a browser window. Sign in with your Azure account, then come back to the terminal.

Confirm it picked the right subscription:

powershellaz account show --query "{Name:name, ID:id}" -o table

If you have multiple subscriptions and it picked the wrong one:

powershellaz account set --subscription "Your Subscription Name"

6 Provision Azure infrastructure

This single script creates everything in Azure and writes your .env automatically. You never need to touch the Azure Portal.

What it createsDefault name
Resource groupbluchip-kg-rg
Cosmos DB account (Gremlin API)bluchip-kg-db
Gremlin databaseBluChipKG
Gremlin graph container (partition key: /pk)BluChip_360
.env file with credentialsauto-written
 
powershellpython setup_azure.py
⏳ Cosmos DB account provisioning takes 3–5 minutes. The script will say "This takes 3–5 minutes, please wait…" — just let it run.

Optional — use a different region or account name:

powershellpython setup_azure.py --location centralindia --account-name my-bluchip-kg

Available regions near India: centralindia  ·  southindia  ·  westindia

When it finishes you'll see:

outputAll done! Your graph is ready.
Next step: python -m src.graph_builder --members 20

11 Query via Python

Open a Python shell with the venv active:

powershellpython
pythonfrom src.utils.cosmos_graph_traversal import BluChipGraphDB

db = BluChipGraphDB()

# High-level graph snapshot
print(db.schema())

# All tiers — earn rates, thresholds, live member counts
print(db.tier_overview())

# All available add-ons
print(db.addon_catalog())

# Full 360 profile of one member
print(db.member_profile("IB45821937"))

# All bookings for a member (latest 10)
print(db.member_bookings("IB45821937", limit=10))

# Segments + add-ons inside one booking
print(db.booking_details("BK12345678"))

# All flight segments this member ever flew
print(db.segment_history("IB45821937"))

# What add-ons does this member buy?
print(db.member_addons("IB45821937"))

# Partner earn history for a member
print(db.member_partner_transactions("IB45821937"))

# Who flies DEL → BOM?
print(db.route_travelers("DEL", "BOM"))

# Add-on + fare breakdown on a route
print(db.segment_purchase_summary("DEL", "BOM"))

# Who are the high-value members?
print(db.high_value_members("HIGH_FREQ_HIGH_SPEND"))

# Members who buy XL seats
print(db.members_by_addon("XL_SEAT"))
